## what i learned

- github pages auto-deploys any repo named `username.github.io` from root, no config needed
- naming conventions matter: no spaces, lowercase, hyphens. repo had to be public or pages won't deploy.
- markdown is the standard format for everything in the class. logs, readmes, briefs, all of it.
- briefs beat prompts. one-shotting "build me a website" gets you slop. a detailed brief with type system, copy rules, references, and structure gets you something usable in one shot.
- HEIC is not supported in browsers.

## how i used ai

- claude (the chat app) for thinking through the design brief, structuring the site, and pulling content from my resume + linkedin into a coherent voice
- claude code for actually building the html and css from the brief, plus all the git operations
- did not use github copilot or vs code at any point

## mistakes / things that broke

- first repo i made was set to private. flipped to public.
- claude code initially struggled with `git push`: kept hitting "RPC failed; HTTP 400" errors. turned out to be a known bug with macos git + http/2 + github. forced http/1.1 and it worked.
- one of my images was a HEIC file (iphone format). browsers don't render HEIC. removed it.
