## what i learned in class

### session 06 (mon 6/8)

- flexbox in depth. container + items model. three main properties on the container: flex-direction (row or column), justify-content (how items space along the main axis), align-items (how items align on the cross axis). flex-direction flips which axis is which, which is why the naming is confusing.
- grid in depth. for two-dimensional layouts. grid-template-columns with fr units lets you split a row into fractional parts. one fr means one share, three fr means three shares. you can mix: `1fr 3fr 1fr` makes a wide middle column.
- prof gave out an openrouter api key for deepseek v4 flash so people can use vs code's agent mode without paying for copilot pro. i don't use vs code or copilot so i didn't need this, but it's a useful path for anyone stuck on the free copilot's 2000 monthly autocompletions.
- key takeaway: layout work used to be done with html tables. modern css gives you flex for one-dimensional alignment and grid for two-dimensional layouts. don't reach for tables.

### session 07 (wed 6/10)

- responsive design. mobile-first as the default. the viewport meta tag is what makes mobile rendering work at all. without it, mobile browsers pretend the screen is 980px wide and zoom out.
- media queries: `@media (min-width: 768px) { ... }` lets you apply different css rules at different screen widths. mobile-first means writing base styles for small screens and adding overrides for larger ones.
- relative units: em and rem scale with font size, percentages scale with parent width. pixels are absolute. for responsive work, prefer relative units.
- forms. the action attribute is the url that receives the form data. the method attribute is get or post. get puts the data in the url (visible, saveable in history, size-limited). post puts the data in the request body (hidden, no size limit, won't end up in browser history).
- input types matter for mobile keyboards. type="email" brings up the @ key. type="tel" brings up the number pad. type="date" gives you a date picker.
- accessibility on forms means using `<label>` properly. clicking a label focuses its input. screen readers announce the label when reading the input.
