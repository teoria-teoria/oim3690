## what i learned in class

### session 10 (mon 6/22)

- the to-do app code from ai looks almost identical across students if you give it the same prompt. one prompt gets you an average product. you have to push past it to make it your own.
- const vs let. const means the binding can't be reassigned. but if the value is an array or object, you can still mutate what's inside. that was confusing at first.
- var is the old keyword. don't use it.
- when you split javascript into its own file you reference it with `<script src="script.js"></script>` at the bottom of body. one file is way cleaner than inline.

### session 11 (wed 6/24)

- data types: string, number, boolean, object, array. arrays are technically objects in javascript, which is weird.
- template literals with backticks let you drop variables into strings using `${variable}`. way cleaner than `"hello " + name + "."`.
- the string-number trap. user input from a form is always a string. `"5" + 3` is `"53"`. you have to parseFloat or Number() before doing math.
- return vs console.log. return hands the value back to whoever called the function. console.log just prints to the dev tools for you to read. if you want to use the result later, you need return.
- arrow functions. `const add = (a, b) => a + b;` is the same as a normal function but shorter. ai uses these a lot.

## mp2 reflection

the personal feature is the tap tempo input. you tap three times (address, top, impact) and the app calculates your current backswing and downswing from the gaps and applies it. pros are nice for reference but most people don't know what tempo they actually have. tapping is the fastest way to set a baseline you can practice against.

design-wise i kept it minimal. one font, one accent color, no gradients, no rounded-everything, no centered hero with two buttons. zhi's ai-look checklist was in my head the whole time. the bpm number is the focal point. everything else supports it.

what i would add: a way to record your tempo over time and chart whether it stabilizes. that would need a real database so it's probably mp3 territory.
