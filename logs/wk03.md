## what i learned in class (session 05)

session 05 was mostly group discussion + intro to css layout. main things i picked up:

- there are three rough buckets for what a website can be. pure front-end static site, front-end with some js for interactivity, or full stack with a backend and database. mp1 only needs the first one. backend isn't graded for this project.
- the box model. every element has content, padding, border, margin. you can see it in devtools. setting `* { margin: 0; padding: 0; box-sizing: border-box }` at the top of your css is a common reset that makes layout less of a fight.
- by default, most elements are block level (take the whole row) but some are inline (sit in the text flow). images are inline by default which is why centering them is annoying. set `display: block` and `margin: 0 auto` and it works.
- flexbox and grid are coming monday. the prof teased a screenshot of apple's 1996 website that used html tables for layout. that's how it used to be done. wild.
